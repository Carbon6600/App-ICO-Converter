# Starlink App ICO Converter 📡🖼️

Простий, швидкий та повністю клієнтський (pure client-side) веб-інструмент для конвертації зображень у багатошаровий формат `.ico` з підтримкою прозорості. Створено спеціально для генерації іконок додатків (зокрема для десктопних утиліт моніторингу Starlink).

👉 **[Жива демо-сторінка (GitHub Pages)](#)** *(заміни це посилання на свій лінк після публікації)*

---

## 🚀 Особливості

* **Мульти-шаровий ICO:** Автоматично пакує зображення у 4 стандартні шари: `16x16`, `32x32`, `48x48` та `256x256` пікселів.
* **Підтримка прозорості:** Зберігає альфа-канал оригінального PNG завдяки пакуванню стиснутих PNG-структур безпосередньо в ICO контейнер.
* **Інтерактивний прев'ю-генератор:** Шаховий фон (checkerboard) дозволяє наочно оцінити, як іконка виглядатиме в різних розмірах до скачування.
* **100% Client-Side:** Обробка файлів, ресайз (`Canvas API`) та бінарне збирання файлу (`DataView`/`ArrayBuffer`) відбуваються прямо у браузері. Жодних серверів чи сторонніх API — ваші вихідні зображення нікуди не завантажуються.
* **Drag-and-Drop:** Зручне перетягування файлів прямо в область конвертера.

## 🛠️ Як це працює зсередини

Конвертер збирає іконку відповідно до офіційної специфікації формату структури ICO:
1.  **Заголовок (Header):** Оголошується тип файлу (`1` для іконок) та кількість шарів (`4`).
2.  **Каталог зображень (Directory):** Для кожного шару записуються метадані (ширина, висота, палітра кольорів, бітова глибина `32-bit` та зміщення в байтах).
3.  **Бінарний блок:** Зображення оптимізуються за допомогою високоякісного згладжування браузера (`imageSmoothingQuality: 'high'`) та записуються у форматі `Uint8Array` послідовно один за одним.

## 📦 Розгортання на GitHub Pages

Оскільки проект складається з одного автономного файлу `index.html`, його деплой займає менше хвилини:

1.  Створіть публічний репозиторій на GitHub.
2.  Завантажте файл `index.html` в корінь репозиторію.
3.  Перейдіть в **Settings** (Налаштування) -> **Pages**.
4.  У розділі **Build and deployment** оберіть source: `Deploy from a branch`.
5.  Вкажіть гілку `main` (або `master`) та папку `/ (root)`, після чого натисніть **Save**.
6.  Через 1-2 хвилини ваша сторінка буде доступна за адресою `https://<ваше-ім'я-користувача>.github.io/<назва-репозиторію>/`.
